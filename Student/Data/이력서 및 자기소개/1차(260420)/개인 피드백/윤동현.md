# 윤동현_자기소개서

## 종합 판단
데이터 테이블과 밸런싱 관점이 좋습니다. 다만 문서 초반의 감정적 게임 경험이 길고, 후반의 강한 시스템 기획 사례가 상대적으로 늦게 나옵니다.

## 잘 한 부분
- Bool 컬럼 남발로 테이블이 비대해진 문제를 인식하고 Enum을 활용해 재구축한 경험은 시스템 기획자로서 강한 근거입니다.
- 데이터 테이블이 부재한 상황에서 직접 설계와 관리를 자원했다는 점은 주도성이 보입니다.
- 밸런싱 전용 빌드를 메인 빌드에서 분리해 QA와 밸런스 피드백 루프를 단축시킨 점은 좋은 실무형 사례입니다.

## 개선 방향 제안
- “게임은 또 하나의 삶” 같은 감정적 지원동기는 길게 쓰면 약해질 수 있습니다. 데이터 구조를 바라보는 본인의 관점과 행동 사례를 초반에 배치해보는 편이 좋습니다.
- 강점과 약점이 데이터 구조 중심으로 좋게 연결되지만, 희망 직무를 시스템/밸런스 기획자로 더 명확히 선언해보는 방향을 제안합니다.
- 결과 표현이 “크게 단축”처럼 다소 추상적입니다. 가능하다면 피드백 흐름이나 테스트 과정이 어떻게 달라졌는지 함께 적어보면 더 좋습니다.
