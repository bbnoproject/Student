# 학생 관리 사이트 페이지 기획

## 1. 문서 목적

이 문서는 학생 데이터 관리 사이트의 `메인 로비`와 `학생 페이지`를 실제 화면 단위로 설계하기 위한 페이지 기획 문서이다.

기존 문서가 데이터 구조와 타임라인 개념을 설명하는 문서였다면, 이 문서는 아래 질문에 답하는 데 집중한다.

- 첫 화면에서 운영진이 무엇을 가장 먼저 봐야 하는가
- 학생을 어떤 방식으로 검색하고 선택해야 하는가
- 학생 상세 화면에서 어떤 순서로 정보를 읽게 할 것인가
- 학생의 현재 상태, 성장 흐름, 위험 신호를 어떤 형태로 시각화할 것인가

---

## 2. 전체 페이지 구조

초기 MVP 기준 페이지 구조는 아래와 같다.

1. 메인 로비
2. 학생 페이지
3. 학생 페이지 > 대시보드 탭
4. 학생 페이지 > 상세정보 탭

향후 확장 가능 페이지는 아래와 같다.

- 학생 페이지 > 원본 기록 탭
- 학생 페이지 > 평가/성향 탭
- 학생 페이지 > 취업 문서 탭

---

## 3. 설계 원칙

### 3.1 학생을 `줄세우기`보다 `해석` 중심으로 본다

메인 로비에서 상위권/하위권을 볼 수 있어야 하지만, 단순 총점 순위만으로 학생을 판단하는 구조는 피한다.

권장 표현 방식은 아래와 같다.

- `종합 우수 학생`
- `성장 폭이 큰 학생`
- `집중 지원 필요 학생`
- `출결/참여 위험 학생`
- `협업 강점 학생`

즉, `순위`는 제공하되 `의미 있는 분류`와 함께 제공한다.

### 3.2 총점보다 `프로파일 + 시간 흐름`을 우선한다

학생은 하나의 점수로 환원하기보다 아래 두 축으로 읽도록 설계한다.

- 현재 상태 프로파일
- 모집부터 현재까지의 변화 흐름

### 3.3 학생 페이지 상단은 `빠른 파악` 전용이다

상단 1/6 이하 영역에서는 운영자가 학생의 핵심 상태를 `5초 안에` 파악할 수 있어야 한다.

### 3.4 상세 정보는 `마일스톤 단위`로 정리한다

원본 기록을 날짜순으로 길게 보여주기보다, 운영 해석이 쉬운 `마일스톤 구간` 단위로 묶는다.

## 4. 메인 로비 기획

## 4.1 페이지 목적

메인 로비는 전체 학생을 종합해서 보는 페이지이다.

이 화면에서 운영진은 아래를 빠르게 판단할 수 있어야 한다.

- 현재 과정 전체 분위기와 위험도
- 상위권/하위권 학생의 분포
- 최근 성장 폭이 큰 학생
- 집중 관찰이 필요한 학생
- 특정 과정/기수/상태의 학생 검색 및 진입

## 4.2 핵심 사용자 행동

1. 학생 검색
2. 학생 필터링
3. 학생군 파악
4. 특정 학생 선택
5. 학생 상세 페이지 진입

## 4.3 화면 구성

### A. 상단 헤더

- 페이지명
- 현재 과정/기수 선택
- 전체 학생 수
- 현재 경고/주의 학생 수
- 최근 업데이트 시점

### B. 검색 및 필터 영역

- 이름 검색
- 과정 필터
- 상태 필터
- 프로젝트 단계 필터
- 출결 위험도 필터
- 성장도 필터

### C. 전체 요약 대시보드 영역

권장 요약 카드

- 전체 학생 수
- 안정 학생 수
- 주의 학생 수
- 경고 학생 수
- 최근 30일 성장 우수 학생 수
- 취업 문서 진행 학생 수

### D. 학생군 분석 영역

메인 로비의 핵심 영역이다.

권장 블록은 아래와 같다.

- `종합 우수 학생 Top N`
- `성장 우수 학생 Top N`
- `집중 지원 필요 학생 N`
- `출결 위험 학생 N`
- `협업 강점 학생 N`

### E. 학생 목록 영역

학생 카드 또는 표 형태로 표시한다.

학생 카드 표시 정보 권장안

- 이름
- 현재 상태 배지
- 현재 프로젝트 단계
- 최근 성장도
- 핵심 강점 1개
- 핵심 주의 포인트 1개

## 4.4 메인 로비의 상위권/하위권 판단 기준

메인 로비에서는 아래 4종 판단 기준을 분리해서 보여주는 것을 권장한다.

### 1. 종합 우수

현재 프로파일이 안정적이고, 출결/참여/협업/성찰 흐름이 고르게 좋은 학생

### 2. 성장 우수

초기 대비 최근 스냅샷 상승 폭이 큰 학생

### 3. 집중 지원 필요

현재 상태가 `주의/경고`이고, 최근 마일스톤에서 위험 신호가 누적된 학생

### 4. 특정 영역 하위

예시

- 자기조절 하위
- 참여 지속성 하위
- 협업 하위
- 진로 목적성 하위

즉, `전체 하위권` 하나보다 `영역별 하위군`이 더 운영 친화적이다.

## 4.5 메인 로비 시각화 권장안

권장 표현 방식은 아래와 같다.

- `학생 분포 산점도`
  - x축: 성장도
  - y축: 현재 종합 상태
  - 목적: “성장 중이지만 아직 불안정한 학생” 같은 군집을 보기 좋음

- `상태 분포 막대 그래프`
  - 안정/주의/경고 학생 수

- `상위 학생 리스트 카드`
  - 이름, 근거 지표, 최근 변화

- `주의 학생 리스트 카드`
  - 이름, 위험 사유, 최근 개입 시점

## 5. 학생 페이지 기획

## 5.1 페이지 목적

학생 페이지는 선택한 학생 1명에 대한 상세 해석 페이지이다.

이 화면에서 운영진은 아래를 파악할 수 있어야 한다.

- 이 학생은 어떤 배경을 가진 학생인가
- 현재 어떤 상태인가
- 시간 흐름상 언제 성장했고 언제 흔들렸는가
- 프로젝트/면담/출결/취업 문서 흐름은 어떤가
- 지금 개입해야 하는가, 관찰만 하면 되는가

## 5.2 상단 레이아웃

상단은 화면 전체 높이의 `1/6 이하` 범위에 배치한다.

### 좌측: 학생 기본 정보

표시 항목

- 이름
- 성별
- 생년월일
- 거주지역
- 연락처
- 학력
- 과정/기수

원칙

- 변경되지 않는 기본 정보만 표시
- 개인정보는 읽기 쉽되 과도하게 넓지 않게 정리
- 민감도가 높은 정보는 필요 시 마스킹 옵션 고려

### 우측: 학생 종합 프로파일

표현 방식은 `n각형 레이더 차트`를 권장한다.

이유는 아래와 같다.

- 6개 평가 준거를 한 번에 읽기 쉽다
- 강점과 약점의 균형을 직관적으로 볼 수 있다
- 학생 간 비교나 시점별 비교로 확장하기 쉽다

표시 축

- 학습 자기조절
- 학습 참여 및 지속성
- 협업 및 관계 형성
- 도전 대응 및 회복탄력성
- 성찰 및 피드백 활용
- 진로 목적성 및 전문성 형성

레이더 차트 아래 보조 텍스트 권장안

- 현재 상태
- 최근 업데이트일
- 대표 강점 2개
- 대표 주의 영역 1~2개

## 5.3 상단 메뉴

상단 기본 메뉴는 탭 구조로 설계한다.

MVP 탭 구성 권장안

1. 대시보드
2. 상세정보

확장 탭 후보

- 원본 기록
- 평가/성향
- 취업 문서

## 6. 학생 페이지 > 대시보드 탭

## 6.1 페이지 목적

대시보드 탭은 `학생 한 명의 요약 해석 화면`이다.

운영자가 이 탭만 보고도 아래를 알 수 있어야 한다.

- 전체 성장 흐름
- 최근 이슈
- 마일스톤별 변화
- 현재 강점/약점

## 6.2 상단 요약 카드

대시보드 상단에는 레이더 차트 아래 또는 옆에 아래 카드들을 둔다.

- 현재 상태
- 최근 면담일
- 프로젝트 제출률
- 출결 위험도
- 반복 협업 팀원 수
- 취업 문서 진행 단계

## 6.3 메인 타임라인 그래프

대시보드의 중심은 `가로형 시간선 그래프`이다.

### 구조

- 가로축: 시간선
- 하단: 전체 커리큘럼
- 상단: 학생 개인 이벤트 요약

### 하단 커리큘럼 레이어

모든 학생에게 동일한 과정 구조를 보여준다.

표시 항목

- 모집
- 개강
- 1차 프로젝트
- 2차 프로젝트
- 3차 프로젝트
- 4차 프로젝트
- 종강
- 주차 또는 주요 수업 모듈

### 상단 학생 이벤트 레이어

학생 개인 데이터에서 추출한 이벤트를 배치한다.

표시 대상

- 모집단계 이벤트
- 출결 이슈
- 프로젝트 팀 배치/역할
- 프로젝트 체크인 및 회고
- 면담
- 취업 문서 작성/피드백
- 성향 스냅샷

### 상호작용

- 이벤트 클릭 시 요약 카드 또는 우측 상세 패널
- 이벤트 유형별 필터
- 마일스톤 구간 강조

## 6.4 성장 스토리 영역

타임라인 아래에는 학생의 `성장 스토리`를 배치한다.

이 영역은 숫자만이 아니라 `운영 해석`이 함께 보여야 한다.

권장 표현 방식

- 마일스톤 카드 6개
- 각 카드마다 성장 곡선 + 핵심 사건 + 해석 문장

## 6.5 마일스톤 구조

사용자 요청 문장에는 `총 5단계`라고 되어 있지만, 실제로 나열된 구간은 `6개`이다.  
따라서 설계는 `6개 마일스톤` 기준으로 정리한다.

마일스톤 구간

1. 모집 ~ 개강
2. 개강 ~ 1차 프로젝트 종료
3. 1차 프로젝트 종료 ~ 2차 프로젝트 종료
4. 2차 프로젝트 종료 ~ 3차 프로젝트 종료
5. 3차 프로젝트 종료 ~ 4차 프로젝트 종료
6. 4차 프로젝트 종료 ~ 종강

## 6.6 마일스톤 성장 곡선 표현

권장 표현 방식은 아래 조합이다.

### A. 구간별 성장 선 그래프

- x축: 마일스톤 6개
- y축: 성장 지수 또는 종합 상태 점수

용도

- 전체 상승/정체/하락 흐름 파악

### B. 영역별 미니 스파크라인

6개 준거 각각에 대해 작은 선 그래프 표시

용도

- “협업은 올랐지만 자기조절은 정체” 같은 패턴 확인

### C. 성장 스토리 서술 카드

각 마일스톤마다 아래를 서술

- 핵심 사건
- 변화 요약
- 강점 발현
- 주의 신호
- 다음 마일스톤 관찰 포인트

## 6.7 학생의 기본 능력치와 성향 표현 방식 추천

추천 조합은 아래와 같다.

### 1. 레이더 차트

용도

- 현재 전체 프로파일 한눈에 파악

### 2. 강점/주의 카드

용도

- 운영자가 바로 액션 포인트 확인

표시 예시

- 강점: 협업, 성찰
- 주의: 자기조절, 참여 지속성

### 3. 마일스톤 성장 선 그래프

용도

- 시간에 따른 변화 확인

### 4. 요약 태그

용도

- 빠른 읽기

예시

- `팀장 경험 있음`
- `회고 강점`
- `출결 안정`
- `진로 문서 진행`

즉, `레이더 차트 + 강점/주의 카드 + 마일스톤 성장선` 조합을 추천한다.

## 7. 학생 페이지 > 상세정보 탭

## 7.1 페이지 목적

상세정보 탭은 `마일스톤별 특이사항과 히스토리 정리`에 초점을 둔다.

대시보드가 요약 화면이라면, 상세정보는 운영자가 근거를 읽는 화면이다.

## 7.2 구성 방식

마일스톤 단위 아코디언 또는 세로 섹션 구조를 권장한다.

각 마일스톤 섹션 안에 아래 내용을 둔다.

- 구간 요약
- 특이사항
- 주요 사건 목록
- 프로젝트 역할/팀 맥락
- 면담 요약
- 출결 이슈
- 취업 문서/회고 요약
- 운영 메모

## 7.3 마일스톤별 표시 항목

각 구간에 아래 항목을 공통으로 둔다.

- 기간명
- 상태 요약
- 성장도
- 주요 이벤트 3~5개
- 특이사항 문장 요약
- 개입 여부
- 다음 구간 관찰 포인트

## 7.4 히스토리 표현 방식

권장 표현 방식은 아래와 같다.

- 좌측: 마일스톤 네비게이션
- 우측: 선택 구간 상세

또는

- 세로 타임라인형 카드 목록

카드 안 세부 블록 권장안

- `상태 변화`
- `프로젝트/협업`

## 11. 요약

이 사이트의 페이지 구조는 아래처럼 이해할 수 있다.

1. 메인 로비는 `전체 학생을 해석하는 페이지`
2. 학생 페이지는 `한 학생의 맥락을 읽는 페이지`
3. 대시보드는 `현재 상태와 성장 흐름 요약`
4. 상세정보는 `마일스톤별 근거와 특이사항 정리`

가장 중요한 설계 포인트는 아래 4가지다.

1. 학생을 단일 점수보다 `프로파일`로 본다.
2. 시간 흐름 안에서 `성장`과 `위험`을 함께 본다.
3. 메인 로비는 `학생군 해석`, 학생 페이지는 `개별 맥락 해석`에 집중한다.
4. 마일스톤 단위 정리는 운영자가 실제로 학생을 이해하고 개입하기 쉽게 만든다.
