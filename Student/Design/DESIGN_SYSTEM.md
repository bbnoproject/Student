# 디자인 시스템 기획

## 1. 디자인 방향

이 앱은 공개 홍보용 사이트가 아니라, 장시간 사용하는 내부 운영 도구이다.  
따라서 시각적 화려함보다 정보 가독성, 빠른 작업 흐름, 명확한 상태 표현, 민감 정보 보호를 우선한다.

핵심 방향은 아래와 같다.

- 정보 밀도가 높아도 읽기 쉬워야 한다.
- 학생 상태와 위험 신호가 시각적으로 빠르게 구분되어야 한다.
- 밝은 모드와 다크 모드가 각각 독립적인 색 체계를 가져야 한다.
- 배경색, 보조색, 강조색의 3가지 베이스를 중심으로 전체 UI를 구성한다.
- 민감 정보는 필요 시 마스킹하거나 축약 표시할 수 있어야 한다.

## 2. 테마 구조

각 모드는 아래 3개의 베이스 컬러를 중심으로 확장한다.

1. 배경색
2. 보조색
3. 강조색(브랜드 컬러)

이 3가지 베이스에서 텍스트, 보더, 카드, 태그, 차트 컬러를 파생한다.

## 3. 라이트 모드

### 컨셉

밝고 정돈되어 있으며 문서형 데이터가 잘 읽히는 모드  
차분한 뉴트럴 톤 위에 선명한 강조색을 두어 중요한 상태가 눈에 띄도록 구성한다.

### 권장 베이스 컬러

- 배경색: `#F6F8FC`
- 보조색: `#DCE6F2`
- 강조색: `#1E6FD9`

### 파생 방향

- 메인 텍스트: `#16202B`
- 서브 텍스트: `#5B6B7F`
- 카드 배경: `#FFFFFF`
- 보더: `#C9D5E4`
- 성공: `#1F9D68`
- 경고: `#E6A700`
- 위험: `#D64545`

## 5. 상태 색상 규칙

상태 표현은 라이트/다크 모두 공통 의미를 유지해야 한다.

- 정상/완료: 초록 계열
- 진행중/검토중: 파랑 계열
- 주의: 노랑 또는 주황 계열
- 위험/경고: 빨강 계열
- 비활성/보류: 회색 계열

## 6. 주요 UI 컴포넌트 방향

### 로그인 화면

- 과도한 장식보다 신뢰감과 명확성을 우선
- 로그인 전 이 앱이 내부 전용 도구임을 분명히 안내
- 권한이 없는 경우 후속 행동이 명확해야 함

### 대시보드 카드

- 숫자와 변화 요약이 먼저 보이도록 구성
- 카드 상단에 범주명, 중앙에 핵심 수치, 하단에 보조 설명 배치
- 위험 카드만 강조색이 아닌 경고색/위험색 사용 가능

### 테이블

- 학생 목록은 테이블 중심 구조
- 헤더 고정
- 정렬 가능
- 상태 태그와 작은 요약 지표를 함께 표시
- 행 클릭 시 상세 화면 이동

### 상세 패널

- 학생 상세 화면은 카드형 섹션 분리
- 긴 텍스트 기록은 타임라인 또는 아코디언 구조 고려
- 민감 정보는 항상 전체 노출하지 않고 필요한 화면에서만 표시하는 방식을 검토

### 폼

- 직접 입력 폼은 단계별 분리 또는 섹션 접기 제공
- 필수 항목 명확 표시
- 업로드 검증 오류는 필드 단위로 표시

### 차트

- 대시보드/분석 화면에서 막대, 도넛, 선형 차트 위주 사용
- 컬러 수를 과도하게 늘리지 않고 상태 의미를 유지

### 보안 관련 UI

- 민감 정보 조회 시 마스킹 해제 여부를 제어할 수 있어야 한다.
- 세션 만료 전 재인증 또는 저장 유도 안내를 제공한다.
- 다운로드/내보내기 기능은 위험도를 고려한 경고 UI를 포함할 수 있다.

## 7. 타이포그래피 방향

- 제목은 강한 가독성을 가진 산세리프 계열
- 본문은 장시간 읽기 쉬운 단정한 산세리프 계열
- 숫자 데이터는 표 정렬이 쉬운 숫자 스타일 고려

후보 예시

- 제목: `Pretendard`, `SUIT`
- 본문: `Pretendard`, `Noto Sans KR`
- 숫자 강조: 탭룰러 숫자 지원 폰트 사용

## 8. 레이아웃 원칙

- 데스크톱 우선 설계
- 고정형 사이드바 + 콘텐츠 영역 구조를 우선 검토
- 테이블, 필터, 폼 사용성이 모바일 반응형보다 우선
- 핵심 버튼은 스크롤 위치와 무관하게 빠르게 접근 가능해야 함

## 9. 접근성 원칙

- 텍스트 대비를 충분히 확보한다.
- 색상만으로 상태를 구분하지 않는다.
- 다크 모드에서도 보더와 구획이 명확해야 한다.
- 키보드 포커스 표시가 분명해야 한다.
