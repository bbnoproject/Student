# 정보 구조 및 화면 구조

## 1. 전체 메뉴 구조

### 앱 공통 메뉴

- 대시보드
- 학생 관리
- 모집단계 관리
- 교과 과정 관리
- 데이터 업로드
- 분석 리포트
- 설정
- 로그아웃

## 2. 앱 진입 흐름

### 2.1 실행 흐름

1. 사용자가 로컬 PC에서 앱 실행
2. 로그인 화면 진입
3. Firebase Authentication 로그인
4. 승인된 사용자 여부 확인
5. 권한 확인 후 대시보드 진입

### 2.2 예외 흐름

- 승인되지 않은 계정 로그인 시 접근 거부 화면 표시
- 세션 만료 시 재로그인 유도
- 네트워크 불안정 시 읽기/저장 실패 안내

## 3. 화면 구조

### 3.1 로그인 화면

목적: 내부 승인 사용자만 앱에 접근하도록 인증한다.

주요 구성

- 앱 이름 및 간단한 설명
- 로그인 버튼
- 계정 상태 안내
- 보안 안내 문구

### 3.2 접근 권한 없음 화면

목적: 로그인은 되었지만 권한이 없는 사용자의 접근을 차단한다.

주요 구성

- 접근 불가 안내 문구
- 관리자 문의 정보
- 로그아웃 버튼

### 3.3 대시보드

목적: 운영진이 전체 학생 현황을 한눈에 파악하는 시작 화면

주요 구성

- 전체 학생 수
- 과정별 학생 수
- 모집단계 진행 현황
- 출석 이슈 요약
- 과제 제출률 요약
- 프로젝트 기록 제출 현황
- 면담 및 경고 대상 요약
- 취업 문서 제출 현황

### 3.4 학생 관리

목적: 학생 목록 확인, 검색, 등록, 수정, 상세 조회

주요 구성

- 학생 목록 테이블
- 과정 필터
- 기수/상태 필터
- 검색
- 학생 등록 버튼
- 파일 업로드 버튼

학생 목록 컬럼 예시

- 이름
- 과정
- 지역
- 모집단계 상태
- 출석 상태 요약
- 과제 제출률
- 최근 면담일

### 3.5 학생 상세

목적: 학생 한 명의 전체 이력을 종합 조회

섹션 구성

- 기본 정보
- 학습 타임라인
- 모집단계
- 교과 과정
- 첨부 파일
- 활동 이력

탭 구조 예시

1. 기본 정보
2. 학습 타임라인
3. 모집단계
4. 출석
5. 과제
6. 프로젝트
7. 면담
8. 취업 문서

학습 타임라인 탭 구성 예시

- 전체 커리큘럼 시간축
- 주차/모듈/프로젝트 구간 표시
- 출결 이벤트 레이어
- 프로젝트 이벤트 레이어
- 면담 이벤트 레이어
- 성향 스냅샷 레이어
- 우측 상세 패널

### 3.6 모집단계 관리

목적: 지원자/학생의 선발 과정을 기록하고 평가

구성

- 제출 서류 관리
- 면접 점수 입력
- 면접 평가 텍스트 입력
- 면접 진행자 기록
- 모집단계 상태 변경

### 3.7 교과 과정 관리

목적: 수강 이후의 운영 데이터를 기록 및 관리

구성

- 출석 이슈 관리
- 과제 제출 현황 관리
- 프로젝트 기록 관리
- 면담 기록 관리
- 취업 문서 관리

### 3.8 데이터 업로드

목적: 다수 학생 데이터를 파일로 일괄 등록/수정

구성

- 파일 업로드 영역
- 업로드 양식 안내
- 컬럼 매핑
- 미리보기
- 오류 검증 결과
- 반영 실행

### 3.9 분석 리포트

목적: 누적 데이터를 기반으로 운영 판단을 돕는 시각화 및 요약 제공

구성

- 과정별 비교
- 기간별 출석 추이
- 과제 제출률 비교
- 면담 빈도 및 경고 현황
- 취업 문서 제출률
- 위험 학생 목록

### 3.10 설정

목적: 운영 기준, 권한, 공통 코드, 보안 정책 관련 기본값 관리

구성

- 과정 목록 관리
- 면접 평가 항목 관리
- 출석 사유 코드 관리
- 사용자 권한 관리
- 테마 설정
- 세션 정책 설정

## 4. 정보 흐름

### 직접 입력 흐름

1. 운영자가 학생 등록 화면 진입
2. 기본 정보 입력
3. 모집단계 또는 교과 과정 정보 추가 입력
4. 저장 후 학생 상세 화면에서 전체 기록 확인

### 파일 업로드 흐름

1. 운영자가 업로드 화면 진입
2. 파일 업로드
3. 컬럼 자동 인식 및 수동 매핑
4. 유효성 검사
5. 오류 데이터 수정 또는 제외
6. 저장 반영
7. 반영 결과 리포트 확인

## 5. 학생 상세 정보의 큰 분류

### A. 기본 정보

- 이름
- 생년월일
- 지역
- 학력
- 연락처
- 이메일
- 신청 과정
- 기수
- 상태값

### B. 모집단계

- 제출 서류
- 자기소개 및 목표
- 면접 점수
- 면접 평가 의견
- 면접 진행자
- 모집단계 결과

### C. 교과 과정

- 출석 이슈
- 과제 제출률 및 평가
- 프로젝트 기록
- 면담 기록
- 취업 문서

## 6. 향후 확장 고려

- 카테고리 추가가 가능해야 한다.
- 각 카테고리 내 세부 항목은 설정 기반으로 늘어날 수 있어야 한다.
- 학생당 복수 과정 이력 또는 과정 변경 이력이 필요할 수 있다.
- 분석 화면은 단순 조회를 넘어 경고 조건 기반 자동 분류로 확장될 수 있다.
- 권한에 따라 보이는 데이터 범위가 달라질 수 있다.
