# 학생 데이터 관리 및 분석 앱 기획서

## 1. 프로젝트 개요

본 프로젝트는 학생 데이터를 수집, 저장, 조회, 분석하여 운영진이 학생 현황을 빠르게 파악하고 의사결정을 내릴 수 있도록 돕는 내부 전용 관리 앱이다.

초기 운영 방식은 공개 웹사이트가 아니라 `로컬에서 실행하는 설치형 앱`을 기준으로 한다.  
사용자는 승인된 PC에서 앱을 실행하고, `Firebase Authentication`으로 로그인한 뒤, 권한이 확인된 경우에만 학생 데이터에 접근할 수 있어야 한다.

학생 데이터 저장소는 `Firebase`를 사용하며, 운영자는 아래 두 방식으로 데이터를 입력할 수 있다.

- 앱 화면에서 직접 입력
- 파일 업로드를 통한 일괄 등록/수정

현재 학생 정보는 아래의 3개 카테고리를 기준으로 수집한다.

1. 기본 정보
2. 모집단계
3. 교과 과정

향후 운영 정책 변화에 따라 데이터 항목은 추가, 수정, 비활성화될 수 있으므로, 구조는 확장 가능해야 한다.

## 2. 프로젝트 목표

### 핵심 목표

- 학생 정보를 한 곳에 통합 저장한다.
- 학생의 상태를 모집 전형부터 교과 과정까지 흐름 단위로 관리한다.
- 누락, 지연, 위험 신호를 빠르게 확인할 수 있도록 분석 결과를 제공한다.
- 운영진이 직접 입력하는 방식과 파일 업로드 방식을 모두 지원한다.
- 공개 URL 없이 내부 승인 사용자만 접근 가능한 구조를 만든다.
- 민감 정보가 로컬 PC에 과도하게 남지 않도록 저장 방식을 제한한다.
- 밝은 모드와 다크 모드를 모두 제공하여 가독성과 사용성을 높인다.

### 기대 효과

- 학생별 이력 추적이 쉬워진다.
- 출석, 과제, 프로젝트, 면담 등 핵심 운영 데이터를 체계적으로 누적할 수 있다.
- 특정 과정, 특정 기수, 특정 상태의 학생을 빠르게 분류할 수 있다.
- 운영 리스크 학생을 빠르게 탐지하고 대응할 수 있다.
- 공개 배포에 따른 불필요한 노출 위험을 줄일 수 있다.

## 3. 운영 원칙

- 이 앱은 외부 공개 서비스가 아니라 내부 운영 도구로 사용한다.
- 승인된 사용자만 로그인할 수 있어야 한다.
- Firebase 보안 규칙에서 기본 접근을 차단하고 필요한 권한만 허용한다.
- 민감 정보는 로컬 파일, 브라우저 캐시, 임시 저장소에 과도하게 남지 않도록 설계한다.
- 학생 데이터는 수집 목적에 필요한 범위 내에서만 저장한다.

## 4. 사용자 가정

초기 기준 주요 사용자는 아래와 같다.

- 관리자: 전체 데이터 관리, 권한 운영, 설정 관리
- 운영진: 학생 등록, 면접 평가 입력, 교과 과정 기록 관리
- 면접관: 면접 평가 점수 및 의견 입력
- 멘토/담당자: 프로젝트 기록, 면담 기록, 출석 이슈 확인 및 입력

초기 버전은 `Windows 환경에서 실행하는 내부 앱`을 우선 가정한다.

## 6. 기술 방향

### 앱 형태

- `Electron` 기반 데스크톱 앱을 우선 검토한다.
- 내부 사용자에게 설치 파일 형태로 배포한다.
- 1차 운영 대상은 Windows PC로 한정한다.
- 인터넷 연결이 가능한 환경에서 Firebase와 통신한다.

### 데이터 저장

- `Firebase Authentication`: 운영자 로그인 및 권한 식별
- `Cloud Firestore`: 학생 및 운영 데이터 저장
- `Firebase Storage`: 제출 서류 원본 파일, 첨부 자료 저장

### 보안 방향

- Firebase 로그인만으로 충분하다고 가정하지 않는다.
- `Firestore Security Rules`와 `Storage Security Rules`를 반드시 설계한다.
- 승인된 이메일 또는 사용자 UID만 접근 허용한다.
- 역할별 읽기/쓰기 권한을 분리한다.
- 로컬 저장은 최소화하고, 오프라인 영구 저장은 기본적으로 제공하지 않는다.

### 파일 업로드 처리

- CSV, XLSX 등 표 형식 파일 업로드를 우선 검토한다.
- 업로드 시 컬럼 매핑, 미리보기, 유효성 검사, 중복 확인 절차를 제공한다.
- 잘못된 데이터는 반영 전 사용자에게 오류를 안내한다.

- 반응형보다는 데스크톱 사용성을 우선한다.
- 다크 모드와 라이트 모드 전환을 지원한다.
- 민감 정보 접근 통제를 제공한다.
- 입력 데이터 유효성 검증을 수행한다.
- 변경 가능성이 높은 항목에 대한 확장성을 확보한다.
- 운영진이 빠르게 학습할 수 있는 단순한 흐름을 제공한다.
- 로컬 PC에 과도한 데이터를 남기지 않도록 설계한다.
