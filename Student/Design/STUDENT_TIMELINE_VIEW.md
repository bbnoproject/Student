# 학생 타임테이블 뷰 기획

## 1. 문서 목적

이 문서는 학생 상세 화면에서 `전체 커리큘럼 시간축` 위에 학생의 상태, 특이사항, 면담, 프로젝트, 성향 변화를 함께 보여주는 `학생 타임테이블 뷰`를 기획하기 위한 문서이다.

이 화면의 목표는 단순 기록 나열이 아니라 아래를 가능하게 하는 것이다.

- 한 학생이 모집부터 현재까지 어떤 흐름을 거쳐 왔는지 빠르게 파악
- 특정 시점의 수업 맥락 안에서 학생 상태를 해석
- 출결, 프로젝트, 면담, 성향 평가를 시간 순서로 연결
- 운영진이 개입이 필요한 시점을 놓치지 않도록 지원

---

## 2. 핵심 아이디어

학생 상세 페이지에 `커리큘럼 타임테이블`을 중심 레이아웃으로 둔다.  
`수업 시간표.xlsx`의 전체 일정이 기본 뼈대가 되고, 그 위에 학생 데이터가 이벤트 레이어처럼 올라가는 구조를 사용한다.

즉, 화면의 기준축은 아래 순서로 잡는다.

1. 과정 전체 일정
2. 주차/모듈/프로젝트 구간
3. 학생 이벤트
4. 학생 상태 및 성향 스냅샷

---

## 3. 화면 목표

### 운영진이 이 화면에서 바로 알고 싶은 것

- 이 학생은 지금 과정의 어느 구간에 있는가
- 최근 언제부터 상태가 흔들렸는가
- 특정 프로젝트 구간에서 어떤 이슈가 있었는가
- 면담 이후 실제 변화가 있었는가
- 현재 성향 프로파일은 어떤 모습인가
- 지금 개입해야 하는가, 관찰만 하면 되는가

### 이 화면이 해결하려는 기존 문제

- 출결, 면담, 프로젝트, 성향 데이터가 흩어져 있음
- 기록은 많지만 시간 흐름 안에서 연결해서 보기 어려움
- “언제부터 문제였는지” 또는 “어느 수업/프로젝트 시기였는지” 추적이 어려움

---

## 4. 기본 사용자 흐름

1. 운영자가 학생 목록에서 학생 검색
2. 학생 상세 페이지 진입
3. 상단에서 학생 기본 프로필과 현재 상태 확인
4. 메인 영역에서 `학생 타임테이블` 확인
5. 특정 날짜/주차/프로젝트 구간 클릭
6. 그 시기의 출결, 면담, 프로젝트 기록, 성향 스냅샷 확인
7. 필요하면 바로 면담 기록 추가 또는 상태 메모 추가

## 6. 타임테이블 구성 방식

### 6.1 기준 축: 커리큘럼 시간표

`수업 시간표.xlsx`의 `시간표` 시트를 기준으로 전체 과정을 시간축으로 정리한다.

기준 정보 예시

- 개강일
- 종강일
- 날짜
- 교과목
- 교육 타이틀

타임테이블에서는 모든 날짜를 그대로 보여주기보다 아래처럼 요약 계층을 둔다.

- 전체 보기: 월/주차 단위
- 확대 보기: 일 단위
- 프로젝트 보기: 프로젝트 기간 중심 보기

### 6.2 학생 이벤트를 일정 위에 겹치기

학생의 기록은 모두 `event` 단위로 시간축 위에 올린다.

이벤트 기본 속성 예시

- 이벤트 날짜
- 이벤트 유형
- 심각도
- 요약 문구
- 상세 문서 링크
- 관련 수업/프로젝트 구간

## 8. 시각화 방식

### 8.1 기본 표현 규칙

- `점 이벤트`: 특정 날짜에 발생한 사건
- `구간 이벤트`: 프로젝트, 특정 수업 모듈, 장기 상태
- `상태 배경`: 위험 구간, 경고 구간, 집중 관리 구간

### 8.2 색상 의미

- 파랑: 일반 정보, 진행중
- 초록: 안정, 정상, 완료
- 노랑: 주의 필요
- 빨강: 위험, 경고, 즉시 확인 필요
- 회색: 참고 정보, 보조 정보

### 8.3 아이콘 예시

- 면담: 말풍선
- 출결: 시계/경고
- 프로젝트: 체크리스트 또는 깃발
- 성향 스냅샷: 레이더 또는 육각형 아이콘
- 취업 문서: 문서 아이콘

### 8.4 강조 규칙

- 최근 14일 이내 이벤트는 강조
- 경고 면담, 반복 결석, 반복 미제출은 두꺼운 강조 표시
- 성향 급변 시점은 별도 마커 사용

## 9. 학생 상태와 성향 표시 방식

### 9.1 상태 표시

타임테이블에서 말하는 `상태`는 운영용 관리 상태를 뜻한다.

예시

- 안정
- 주의
- 집중 관찰
- 경고
- 취업 준비 집중

상태는 하나의 날에만 붙이기보다 `기간 상태`로 보여주는 것이 적합하다.

예시

- `2026-03-18 ~ 2026-04-03`: 프로젝트 과부하 주의 구간
- `2026-04-10 ~ 2026-04-24`: 면담 후 회복 관찰 구간

### 9.2 성향 표시

성향은 [EVALUATION_CRITERIA.md](C:\Users\안중재\Desktop\PROJECT\Student\Design\EVALUATION_CRITERIA.md)의 6개 영역을 기반으로 한다.

표시 방식은 아래 2단계로 나눈다.

#### A. 현재 성향 요약

상단 카드에 현재 최신 점수 요약

- 자기조절
- 참여 및 지속성
- 협업
- 회복탄력성
- 성찰
- 진로 목적성

#### B. 시점별 성향 스냅샷

주요 구간마다 스냅샷을 저장해 변화를 보여준다.

권장 저장 시점

- 입과 초기
- 1차 프로젝트 종료 후
- 2차 프로젝트 종료 후
- 3차 프로젝트 중간/종료 후
- 정기 면담 시점

타임테이블에서는 각 시점에 `성향 스냅샷 점`을 두고 클릭 시 해당 시점의 6영역 값을 보여준다.

## 14. 운영 관점의 기대 효과

- 한 학생의 전체 맥락을 빠르게 파악할 수 있다.
- 면담 기록을 “언제 어떤 맥락에서 있었는지” 이해할 수 있다.
- 특정 프로젝트가 학생 상태에 어떤 영향을 주었는지 볼 수 있다.
- 성향 평가를 단일 점수가 아닌 변화 흐름으로 볼 수 있다.
- 개입 시점과 개입 후 변화를 추적할 수 있다.

## 16. 요약

학생 타임테이블 뷰는 `학생 상세 페이지의 중심 기능`으로 설계할 가치가 있다.  
이 화면은 단순 기록 목록보다 훨씬 강하게 학생의 맥락을 보여주며, 현재 보유한 데이터와 교육학 기반 평가 준거를 하나로 묶는 핵심 인터페이스가 될 수 있다.

핵심 설계 원칙은 아래와 같다.

1. 시간표를 기준 축으로 사용한다.
2. 학생 이벤트를 같은 축 위에 겹친다.
3. 성향은 고정 라벨이 아니라 시점별 스냅샷으로 보여준다.
4. 총점보다 변화 흐름과 근거를 우선한다.
