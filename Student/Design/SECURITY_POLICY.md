# 보안 정책 및 개인정보 보호 원칙

## 1. 목적

이 문서는 학생의 민감한 개인정보와 운영 데이터를 다루는 내부 앱의 보안 원칙을 정의한다.  
앱이 로컬에서 실행된다고 해서 자동으로 안전해지는 것은 아니므로, 인증, 권한, 저장 정책, 기기 관리까지 포함한 운영 기준을 함께 정한다.

## 2. 기본 원칙

- 기본 접근은 항상 차단하고 필요한 권한만 허용한다.
- 공개 URL 기반 서비스로 운영하지 않는다.
- 승인된 사용자만 로그인할 수 있어야 한다.
- 학생 데이터는 목적 범위 내 최소한으로 저장한다.
- 로컬 기기에는 학생 원본 데이터를 과도하게 남기지 않는다.
- 보안은 앱 UI가 아니라 Firebase 규칙을 기준으로 보장한다.

## 3. 인증 및 사용자 접근 정책

### 인증

- `Firebase Authentication`을 사용한다.
- 운영진 전용 계정만 로그인 가능해야 한다.
- 공용 계정 사용은 금지한다.

### 접근 허용 기준

- 로그인 성공 후에도 `users` 컬렉션 또는 허용 목록에서 승인 여부를 한 번 더 확인한다.
- `active = true`인 사용자만 실제 데이터 접근을 허용한다.
- 역할(`role`)에 따라 기능 범위를 구분한다.

### 역할 예시

- `admin`: 전체 관리, 권한/설정 수정 가능
- `operator`: 학생 정보 입력 및 수정 가능
- `interviewer`: 면접 관련 데이터 중심 접근
- `mentor`: 교과 과정 기록 중심 접근

## 4. Firebase 보안 정책

### Firestore 원칙

- 기본값은 모든 읽기/쓰기를 거부한다.
- 인증된 사용자 중 승인된 계정만 허용한다.
- 컬렉션별로 읽기/쓰기 권한을 분리한다.
- 설정 데이터와 학생 데이터의 권한을 동일하게 두지 않는다.

### Storage 원칙

- 원본 서류와 첨부 파일은 인증된 승인 사용자만 접근 가능해야 한다.
- 업로드/다운로드 권한을 역할별로 분리한다.
- 파일 경로는 학생 ID와 문서 종류 기준으로 통제한다.

## 5. 로컬 기기 보안 원칙

- 앱은 승인된 Windows PC에서만 사용한다.
- 공용 PC 사용은 가급적 피한다.
- OS 로그인 잠금, 화면 잠금, 디스크 암호화 사용을 권장한다.
- 학생 데이터를 별도 로컬 DB에 영구 저장하지 않는다.
- 임시 업로드 파일은 처리 후 삭제하는 흐름을 우선 검토한다.
- 자동 로그인과 과도한 캐시 저장은 최소화한다.

## 6. 앱 동작 보안 원칙

- 장시간 미사용 시 세션 만료 또는 재인증을 요구한다.
- 민감 필드는 기본적으로 부분 마스킹을 검토한다.
- 다운로드, 복사, 내보내기 기능은 기본 비활성 또는 제한적으로 허용한다.
- 권한 없는 사용자는 메뉴 진입 이전에 차단한다.
- 중요한 작업은 감사 로그를 남긴다.

## 7. 업로드 및 첨부 파일 처리 원칙

- 업로드 전에 필수값, 형식, 중복 여부를 검증한다.
- 업로드 파일은 가능한 한 앱 내부 처리 후 장기 보관하지 않는다.
- 첨부 파일은 Storage에 저장하고, 앱 로컬 폴더 보관은 기본적으로 피한다.
- 파일명에 주민번호 등 과도한 개인정보가 포함되지 않도록 관리 기준이 필요하다.

## 8. 로그 및 감사 정책

- 아래 작업은 감사 로그 대상으로 우선 검토한다.

- 학생 신규 등록
- 학생 정보 수정
- 학생 정보 삭제 또는 비활성화
- 파일 업로드
- 권한 변경
- 설정 변경

감사 로그 예시 정보

- 작업자 UID
- 작업자 이메일
- 작업 유형
- 대상 ID
- 작업 시각
- 변경 요약

## 9. 사고 대응 원칙

- 승인되지 않은 계정이 확인되면 즉시 접근을 차단한다.
- 기기 분실 또는 계정 유출이 의심되면 사용자 계정을 비활성화한다.
- 보안 규칙 변경 이력은 별도 관리가 필요하다.
- 민감 데이터 오노출 발생 시 어떤 데이터가 노출되었는지 추적 가능해야 한다.

## 10. 구현 우선순위

1. Firebase 인증 도입
2. 승인 사용자 판별 로직
3. Firestore/Storage 보안 규칙 작성
4. 역할 모델 정의
5. 세션 정책 및 로그아웃 처리
6. 감사 로그 설계
7. 민감 정보 마스킹 및 다운로드 정책
