# 구현 우선순위 및 단계별 계획

## 1. 전제

현재 단계는 구현이 아니라 기획 단계이다.  
아래 로드맵은 이후 실제 개발을 진행할 때의 우선순위를 정리한 것이다.

이번 변경 이후의 핵심 전제는 아래와 같다.

- 공개 웹 배포는 진행하지 않는다.
- 내부 승인 사용자만 사용하는 로컬 설치형 앱으로 운영한다.
- 로그인과 데이터 저장은 Firebase를 유지한다.
- 기능 개발과 같은 비중으로 보안 설계를 초기부터 포함한다.

## 2. 1차 목표(MVP)

가장 먼저 만들어야 하는 최소 기능 집합은 아래와 같다.

### 핵심 범위

- Electron 기반 앱 실행 구조
- 관리자/운영진 로그인
- 승인 사용자 접근 통제
- 학생 기본 정보 등록/수정/조회
- 학생 목록 및 상세 화면
- 모집단계 정보 입력
- 교과 과정의 핵심 입력 기능
- 파일 업로드를 통한 기본 정보 등록
- 기본 대시보드
- 라이트 모드/다크 모드

### MVP에서 우선 포함할 세부 기능

1. 앱 실행 및 로그인 화면
2. 학생 등록 폼
3. 학생 목록 테이블
4. 학생 상세 화면
5. 면접 기록 입력
6. 출석 이슈 입력
7. 과제 제출 현황 입력
8. 면담 기록 입력
9. 취업 문서 제출 여부 입력
10. CSV 업로드
11. 기본 통계 카드
12. Firebase 보안 규칙 초안

## 6. 리스크 및 사전 고려사항

### 로컬 앱 관련

- 설치 파일이 외부로 유출되어도 로그인과 권한 통제가 약하면 데이터는 노출될 수 있다.
- 공용 PC 사용 시 로그아웃, 세션 종료, 화면 잠금 정책이 필요하다.
- 임시 파일과 다운로드 파일이 남는 위치를 통제해야 한다.

### Firebase 관련

- Firebase API 키 비공개 자체보다 보안 규칙이 더 중요하다.
- 인증만 통과하면 모두 읽을 수 있는 구조가 되지 않도록 주의해야 한다.
- Storage 파일 접근 규칙도 Firestore와 별도로 설계해야 한다.

### 데이터 구조 관련

- 초기에는 단순해 보여도, 이후 항목 추가가 많아질 가능성이 높다.
- 면접 평가 항목, 출석 사유, 문서 종류 등은 설정형 구조가 유리하다.

### 운영 관련

- 동일 학생 중복 등록 기준을 명확히 해야 한다.
- 파일 업로드 시 식별 키 기준이 필요하다.
- 민감 정보 노출 방지 정책이 필요하다.
- 내보내기 기능은 후순위로 두는 편이 안전하다.
